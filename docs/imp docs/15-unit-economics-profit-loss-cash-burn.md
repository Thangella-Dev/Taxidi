# 15 - Taxiro Unit Economics, Profit, Loss, And Cash Burn

**Date:** 23 July 2026

## What Is Unit Economics?

Unit economics means how much Taxiro earns or loses per ride after direct costs.

## Example 1 - Rs 100 Bike Ride, 7% Commission, Online Payment

| Item | Amount |
|---|---:|
| User fare | Rs 100.00 |
| Taxiro commission at 7% | Rs 7.00 |
| Rider gross earning | Rs 93.00 |
| Payment gateway fee approx 2% + GST | Rs 2.36 |
| SMS/push/support estimated direct cost | Rs 0.50-Rs 2.00 |
| Map/API estimated direct cost | Rs 0.50-Rs 3.00 |
| Taxiro net before incentives/overheads | Rs 1.64-Rs 4.64 |

If Taxiro gives even Rs 10 user coupon or rider incentive, this ride becomes loss-making.

## Example 2 - Rs 200 Ride, 7% Commission, Online Payment

| Item | Amount |
|---|---:|

| Rider gross earning | Rs 186.00 |

Higher fare rides help, but incentives can still burn money quickly.

## Example 3 - Cash Ride

| Item | Amount |
|---|---:|
| User pays rider | Rs 100 |
| Taxiro commission due | Rs 7 |
| Payment gateway fee | Rs 0 |
| Main risk | Taxiro must collect Rs 7 from rider later |

Cash rides look cheaper, but commission collection becomes operationally difficult without rider wallet/ledger.

## Where Money Burns

| Burn area | Why it burns money |
|---|---|
| Rider incentives | Needed to keep supply online when demand is low. |
| User coupons | Needed for acquisition and repeat usage, but reduces margin. |
| Payment gateway fees | Every online payment charges percentage. |
| Maps/search/routing APIs | Every map load/search/route can cost money at scale. |
| SMS/WhatsApp | OTP and emergency notifications cost per message. |
| Support team | Ride issues need fast human response. |
| Refunds/disputes | Failed rides and payment issues create losses. |
| Fraud | Fake rides, fake GPS, cancellation abuse, coupon abuse. |
| KYC | Per rider verification costs money. |
| Legal/compliance | Transport, privacy, payment, and labour compliance. |

## How To Reduce Loss

- Start in one dense area only.
- Keep rider incentives time-bound and data-driven.
- Use referrals instead of broad coupon burning.
- Push online wallet payments gradually.
- Reduce support issues through better UX and clear policies.
- Use demand signals to reduce idle rider time.
- Monitor fraud and fake GPS early.
- Add corporate/campus partnerships for repeat demand.

## Break-Even Thinking

If Taxiro earns Rs 7 on a Rs 100 ride, it needs very high volume or higher monetization to cover real business costs.

Better paths:

- Raise commission after trust is built, maybe 10-15% in some categories.
- Add rider subscription model in high-demand zones.
- Add convenience fee for users only if value is clear.
- Build B2B/corporate partnerships.
- Use wallet to reduce commission leakage.
